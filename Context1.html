A modern Enterprise Data Lake architecture was designed and implemented to unify data pipelines across health programs, operational systems, and mission performance frameworks.

The platform became the foundation for analytics engineering, operational reporting, and national-level health program data aggregation.

Layer 3: Engineering

Metadata-Driven ETL

We removed hardcoded pipelines. New data sources are "configured" not "coded."

{
  "source": "Zambia_Field_Office",
  "type": "MS_Access",
  "schedule": "0 0 * * *",
  "mappings": [
    {"src": "Exp_Amt", "dest": "ExpenseAmount", "cast": "DECIMAL(18,2)"},
    {"src": "Prj_Code", "dest": "ProjectKey", "cast": "INT"}
  ]
}

Lessons Learned

Alignment First

Stakeholder alignment matters more than the tech stack. Buy-in from the field was the true MVP.

Keep it Lean

Over-engineering is dangerous. Simple, robust pipelines beat complex, fragile ones every time.

Model for Growth

Data modeling decisions compound over time. Kimball foundations enabled scaling from 1 to 15 countries.

Early Visibility

Observability must be built early. Knowing where the pipe broke is as important as the flow itself.
